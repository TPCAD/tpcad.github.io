基本知识

标识符

标识符是用来标识变量、函数或其他用户自定义项目的名称。

标识符由大写字母 A-Z 或小写字母 a-z 或下划线 _ 开始，后接零个或多个大小写字母、下划线和数字。标识符严格区分大小写。

下面是一些合法的标识符：_temp、moth、a_234。

下面是一些不合法的标识符：123asd、@month。

注释

单行注释：//。

多行注释：/**/。

注释不允许嵌套，不能出现在字符串或字符值中。

关键字

关键字是 C 的保留字，它们不能被用作常量名、变量名或其他标识符名字。

数据类型

数据类型决定变量在内存中的存储大小以及如何存储。

C 中各种类型的实际存储大小与系统位数有关，可以用 sizeof 运算符来获取某个类型在当前机器的存储大小。（在 GCC 下，sizeof 的返回值是 long 类型）

整数类型

类型 存储大小 值范围
int 2 或 4 字节 -32,76832,767 或 -2,147,483,6482,147,483,647
unsigned int 2 或 4 字节 065,535 或 04,294,967,295
short 2 字节 -32,768~32,767
unsigned short 2 字节 0~65,535
long 4 字节 -2,147,483,648~2,147,483,647
unsigned long 4 字节 0~4,294,967,295

int 类型是有符号整型。C 规定 int 的存储大小至少为 2 字节（现在一般为 4 字节）。

C 通常假定整数常量是十进制数。可以使用特定前缀来表示特定进制。

八进制：0

十六进制：0x 或 0X

除了常见的 int 类型外，还有 long int、long long int。

C 通常把整数常量存储为 int 类型，如果想让其存储为 long 类型，可以在整数常量后加上 l 或 L 后缀。如 88l、89L。类似地，如果想让整数常量存储为 long long 类型，可以在其后加上 ll 或 LL 后缀。如 8999ll、8789LL。

浮点类型

类型 存储大小 值范围 精度
float 4 字节 1.2E-38 ~ 3.4E+38 6 位有效位
double 8 字节 2.3E-308 ~ 1.7E+308 15 位有效位
long double 16 字节 3.4E-4932 ~ 1.1E+4932 19 位有效位

字符类型

char 类型用于存储字符。实际上，char 类型存储的是整数类型。计算机通过编码技术来处理字符，使每一个字符都对应一个整数。C 采用的是 ASCII 编码。

类型 存储大小 值范围
char 1 字节 -128127 或 0255
unsigned char 1 字节 0~255
signed char 1 字节 -128~127

根据编译器的不同，char 的取值范围也会有所不同。如在 GCC 下，char 的取值范围是 -128127，而在有些编译器下，char 的取值范围是 0255。

C 中单引号括起来的字符表示字符常量。

在 ASCII 编码表中，并不是所有字符都是可打印字符，还有一部分非打印字符。

除了使用对应的整数外，还可以使用转移序列来表示这些字符。

关系运算符

假设 A=10, B=20。

运算符 描述 实例
== 检查两个操作数的值是否相等，如果相等则条件为真。 (A == B) 为假。
!= 检查两个操作数的值是否相等，如果不相等则条件为真。 (A != B) 为真。
> 检查左操作数的值是否大于右操作数的值，如果是则条件为真。 (A > B) 为假。
< 检查左操作数的值是否小于右操作数的值，如果是则条件为真。 (A < B) 为真。
>= 检查左操作数的值是否大于或等于右操作数的值，如果是则条件为真。 (A >= B) 为假。
<= 检查左操作数的值是否小于或等于右操作数的值，如果是则条件为真。 (A <= B) 为真。

逻辑运算符

假设 A=1, B=0。

运算符 描述 实例
&& 逻辑与。如果两个操作数都非零，则条件为真。 (A && B) 为假。
|| 逻辑或。如果两个操作数中有任意一个非零，则条件为真。 (A || B) 为真。
! 逻辑非。用来逆转操作数的逻辑状态。如果条件为真则逻辑非运算符将使其为假。 !(A && B) 为真。

逻辑与和逻辑或都有短路性质，即若第一个操作数（表达式）就可以得出结果，那么第二个操作数（表达式）将不会被判断（执行）。

函数

函数是完成特定任务的独立程序代码单元。使用函数可以减少编写重复代码，提高代码利用率。

定义函数

返回值类型：定义函数时需要指定函数的返回值的类型。一个函数可以有零个或一个返回值。若函数没有返回值，则返回值类型定义为 void。

函数名：函数名是函数的实际名称。不允许重复。

参数表：参数是函数在执行过程中需要从函数外部获取的值。定义参数表时必须明确参数的类型，名称，数量，顺序。参数并不是必须的，若函数不需要参数，则参数表定义为 void 或直接留空。

函数主体：完成特定任务的独立程序代码单元。

函数声明

函数声明可以告诉编译器函数名称及如何调用函数。函数声明通常写在预处理之后，主函数之前。

函数声明必须包含函数返回值类型，函数名以及参数表。

在函数声明中，参数类型是必须的，而参数名称不是。

调用函数

传递参数时，参数的类型、数量、顺序必须严格按照函数定义的参数表来传递。

return 语句

当函数有返回值时，就需要用到 return 关键字。

当函数没有返回值时，return 可以立即结束函数。

单向值传递

在 C 中，函数的传值方式是单向值传递。

在执行函数的过程中，程序会创建两个变量 num1、num2，并把它们初始化为对应的实参的值。这一过程便把实参的值传递给了形参。两个形参和其对应的实参只有值是相同的，除此之外没有其他任何联系。所以在函数内修改形参的值并不会影响到实参。

实际上就是将实参复制了一份，函数内的所有操作都实在复制出来的那一份上进行的，与原来的那一份没有关系。

形参在函数调用结束后就会自动删除。

作用域

作用域是变量在所存在的范围。

局部变量

在函数内或者块内声明的变量都是局部变量。局部变量只能作用于其所在的函数内或块内。

系统不会对局部变量进行初始化，所以在使用局部变量时应先对其进行初始化，不然可能会发生意料之外的情况。

全局变量

在所有函数之外声明的变量是全局变量。全局变量能作用于整个程序，且可以被所有函数使用。

系统会自动对全局变量初始化。下表是对应数据类型的初始化值。

数据类型 初始化值
int 0
char ‘\0’
float 0
double 0

形式参数

形式参数的作用范围只在其所在的函数之内。

存储类

存储类决定了变量或函数的范围和声明周期。

C 有四种存储类：

  • auto
  • register
  • static
  • extern

auto

auto 是所有局部变量的默认存储类。auto 也只能修饰局部变量。

register

register 用于定义存储在寄存器中的局部变量。这意味着不能对变量应用 &，因为它没有内存地址。

被 register 修饰并不意味着变量将被存储在寄存器中，它意味着变量可能存储在寄存器中，这取决于硬件和实现的限制。

static

static 修饰全局变量时会使其作用域被限制在当前文件内。

static 修饰局部变量时，会使变量的生命周期与整个程序相同，且只会被初始化一次。static 修饰的局部变量的作用域依然是其所在函数或块内，并不会改变其作用域。

文件操作

文件指针

在 stdio.h 有一个名为 FILE 的结构体，文件指针就是指向 FILE 的指针。

打开/关闭文件

打开文件

使用 fopen( ) 函数打开文件。

fopen( ) 接受两个参数（都是字符串类型），第一个参数为待打开文件的名称，第二个参数为打开模式。

模式字符串 含义
“r” 以读模式打开文件
“w” 以写模式打开文件，把现有文件长度截为 0，如果文件不存在，则创建一个新文件
“a” 以写模式打开文件，在现有文件末尾添加内容，如果文件不存在，则创建一个新文件
“r+” 以更新模式打开文件（即可以读写文件）
“w+” 以更新模式打开文件（即可以读写文件）,把现有文件长度截为 0，如果文件不存在，则创建一个新文件
“a+” 以更新模式打开文件（即可以读写文件），在现有文件末尾添加内容，如果文件不存在，则创建一个新文件；可以读整个文件，但只能在末尾添加内容
“rb”、”wb”、”ab” “r+b”、”w+b”、”a+b” 以二进制模式打开文件
“wx”······ 若文件存在或以独占模式打开文件，则打开文件失败

注意：

  1. 读或写模式只能对文件进行对应操作
  2. 更新模式可以对文件进行读或写操作
  3. “w” 模式打开一个现有文件时，会先把文件内容删除
  4. “x” 模式只用于 “w” 模式
  5. “x” 模式无法打开现有文件

fopen( ) 函数返回 FILE 类型指针，若打开文件失败，会返回 NULL。在打开文件时应该对检查文件是否正确打开。

关闭文件

使用 fclose( ) 函数关闭文件。

fclose( ) 函数接受一个 FILE 类型指针作参数。若关闭成功返回 0，失败则返回 EOF。

在文件使用完毕后应及时关闭文件，并检查是否关闭成功。
